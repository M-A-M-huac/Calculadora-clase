### Pasos para completar la tarea:

1. **Preparar el proyecto para subirlo a GitHub:**
   - Asegúrate de que tu proyecto esté completo y funcionando localmente.
   - Crea un archivo `.gitignore` en la raíz del proyecto y añade las siguientes líneas para ignorar archivos innecesarios:
     ```
     node_modules/
     .env
     ```
   - Inicializa un repositorio Git en tu proyecto:
     ```
     git init
     ```
   - Añade todos los archivos al repositorio:
     ```
     git add .
     ```
   - Realiza un commit inicial:
     ```
     git commit -m "Initial commit"
     ```

2. **Subir el proyecto a GitHub:**
   - Crea un nuevo repositorio en GitHub (sin inicializar con README ni .gitignore).
   - Copia el enlace del repositorio (por ejemplo, `https://github.com/tu-usuario/nombre-repositorio.git`).
   - Conecta tu proyecto local al repositorio remoto:
     ```
     git remote add origin <URL-del-repositorio>
     ```
   - Sube los cambios al repositorio remoto:
     ```
     git branch -M main
     git push -u origin main
     ```

3. **Configurar Render para desplegar el backend:**
   - Crea una cuenta en [Render](https://render.com/) si no tienes una.
   - En el panel de Render, selecciona "New" > "Web Service".
   - Conecta tu cuenta de GitHub y selecciona el repositorio que subiste.
   - Configura el servicio:
     - **Branch:** `main`
     - **Root Directory:** `backend` (si tu backend está en esa carpeta).
     - **Build Command:** `npm install`
     - **Start Command:** `node server.js` (o el comando que uses para iniciar tu servidor).
   - Añade las variables de entorno necesarias (por ejemplo, conexión a MongoDB Atlas) en la sección "Environment".
   - Haz clic en "Create Web Service" y espera a que Render despliegue tu aplicación.

4. **Configurar MongoDB Atlas:**
   - Crea una cuenta en [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) si no tienes una.
   - Crea un nuevo clúster gratuito.
   - Configura una base de datos y colección para tu proyecto.
   - Copia la URL de conexión (Connection String) y configúrala como variable de entorno en Render.

5. **Probar los endpoints con Postman, Insomnia o Thunder Client:**
   - Asegúrate de que tu backend esté desplegado y funcionando en Render.
   - Prueba los tres endpoints (`GET`, `POST`, `DELETE`) usando la URL de tu backend desplegado.
   - Guarda capturas de pantalla de las pruebas exitosas.

6. **Crear el PDF con la documentación:**
   - Crea una portada con el título del proyecto, tu nombre, y la fecha.
   - Incluye los siguientes elementos:
     - **Link de Render:** La URL de tu backend desplegado.
     - **Link de GitHub:** La URL de tu repositorio.
     - **Capturas de pantalla:**
       - Repositorio en GitHub.
       - Servicio desplegado en Render.
       - Base de datos en MongoDB Atlas mostrando la colección.
       - Pruebas de los endpoints en Postman, Insomnia o Thunder Client.

7. **Entregar el PDF:**
   - Asegúrate de que el PDF esté completo y bien organizado.
   - Entrega el archivo según las instrucciones de tu profesor.
